## 3. The "Pain" Log (CRITICAL)

### Bug: Silent Skill Installation Failure
* **Tried:** Installing plugin via `setup-api-key.sh`
* **Failed:** Skills never appeared because script referenced lowercase `skill.md` but files are `SKILL.md` (macOS case-insensitive FS masked this on dev, would break on Linux CI)
* **Root Cause:** Git case rename (commit `84fe59f`) changed files but `setup-api-key.sh` wasn't updated
* **Fix:** Updated all 4 references to `SKILL.md`
* **Note:** Always verify case-sensitive paths after git renames. Linux CI will catch what macOS hides.

### Bug: install_agents.sh Creates Broken Structure
* **Tried:** Running `./install_agents.sh` from repo root
* **Failed:** Created `letter-for-my-future-self/` nested directory with shell-syntax SKILL.md files (not valid Claude skills) and version 1.0.0
* **Root Cause:** Legacy bootstrapper from initial commit, never updated as plugin structure evolved
* **Fix:** Complete rewrite — now copies actual source files to `~/.claude/agents/` and `~/.claude/skills/`

### Security: Shell Injection in API Key Setup
* **Tried:** `setup-api-key.sh` writes user input directly into JSON via heredoc
* **Failed:** Special characters (`"`, `$`, backticks) in API keys break JSON or cause shell expansion
* **Fix:** Replaced `cat <<EOF` with `printf '%s'` for safe escaping
* **Note:** Consider migrating to `jq` for proper JSON construction if complexity grows

### Security: API Key Exposure via Masking
* **Tried:** `show_config_status()` displays `key[-4:]` for verification
* **Failed:** Keys shorter than 4 chars would display in full
* **Fix:** `_mask_key()` helper requires `len(key) >= 8` before showing last 4 chars, otherwise shows `****`

## 4. Architecture Analysis: Cross-Project Integration

### nyxCore Memory Consumption (42 files, 12,263 lines, Feb-Mar 2026)
nyxCore is the primary consumer of this plugin's output. It reads `.memory/letter_*.md` files and feeds them into:
1. **Workflow Engine** — template variable `{{memory}}` injects curated workflow insights
2. **Discussion Context** — 8 wisdom sources loaded, including session memory
3. **Blog Pipeline** — GitHub Actions → Anthropic API → PR with draft
4. **Consolidation Service** — cross-project pattern extraction from memory

### Token Economics (measured from nyxCore usage patterns)
* Average letter file: ~800-1,500 tokens
* Blog generation per letter: ~$0.03 (Sonnet 4)
* 42 letters over 28 days = ~1.5 letters/day average
* Estimated monthly blog cost: ~$1.35/month at current rate

### Filename Format Impact
The new `letter_YYYYMMDDHHMMSS.md` format provides:
* **No counter collisions** — multiple sessions same day produce unique files
* **Precise chronology** — second-level granularity for temporal queries
* **Simpler sort** — single 14-digit key vs compound date_counter
* **Backwards compatible** — `get_memory_file()` handles all three legacy formats

### Recommended Next Steps for nyxCore Integration
1. **PreCompact hook** — Auto-checkpoint before context compaction (prevents the #1 data loss scenario)
2. **Session metadata section** — Add duration, files touched, commits made to letter template
3. **Rolling summary compaction** — `/letter-compact` skill using Haiku ($1/MTok) to summarize old letters
4. **Cross-project pain index** — Global `~/.config/letter-for-my-future-self/pain_index.json` aggregating pain items across projects for proactive warnings
